# Report Prototyping Workflow

This document outlines a practical workflow for using AI to support Power BI report prototyping.

## Step 1: Define the audience
Determine who will use the report: executives, managers, analysts, or operational teams.

## Step 2: Clarify the business questions
List the decisions the report should support.

## Step 3: Identify the core KPIs
Prioritize the top metrics users need to see first.

## Step 4: Draft the layout
Sketch KPI cards, trend views, detail tables, and filter behavior.

## Step 5: Use AI for refinement
Use AI to suggest:
- dashboard structure
- section labels
- KPI wording
- visual hierarchy
- simplified layout options

## Step 6: Build a first version
Create a prototype in Power BI using sample or masked data.

## Step 7: Validate with users
Confirm that the design is understandable and supports the intended decisions.

## Step 8: Refine and simplify
Improve labels, logic, drill paths, and documentation based on feedback.

## Deliverables

- dashboard wireframe
- KPI list
- report specification
- validation notes
